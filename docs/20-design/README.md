# Wave UX — Design System Planning

## Purpose

Wave UX planning captures the premium visual identity and interaction system that Business OS will adopt during the Wave UX implementation program (UX-1 through UX-5). This wave is **planning-only**; no application code changes are made here.

## Scope Boundary

- Documentation only. No changes to `src/**` or `src/styles.css`.
- Consumed by the implementation waves UX-1 through UX-5, sequenced after Wave 0.5 exit.

## Reading Order

1. `UX_PRINCIPLES.md` — the "why".
2. `DESIGN_SYSTEM.md` — brand language.
3. `DESIGN_TOKENS.md` — token architecture that maps to Tailwind v4.
4. `COMPONENT_GUIDELINES.md` — component catalog.
5. `LAYOUT_STANDARD.md` — canonical layouts.
6. `ENTERPRISE_UX_PATTERNS.md` — dense-data patterns.
7. `ACCESSIBILITY_STANDARD.md` — WCAG 2.2 AA operating rules.
8. `RESPONSIVE_STANDARD.md` — breakpoints and adaptation.
9. `DASHBOARD_EXPERIENCE.md` — dashboard system.
10. `MOBILE_ALIGNMENT.md` — cross-platform parity.
11. `UX_IMPLEMENTATION_ROADMAP.md` — UX-1 through UX-5 sequencing.

## Ownership

- **Design Council** — visual system, tokens, components.
- **Platform** — implementation waves UX-1 through UX-5.
- **Accessibility Lead** — WCAG conformance.

## Governance

Wave UX planning documents follow `STANDARDS_LIFECYCLE_STANDARD.md` frontmatter conventions.
